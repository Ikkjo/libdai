\doxysubsection{Detailed Description}
\subsubsection*{template$<$\mbox{\hyperlink{namespacedai_a94b4dd96324af3c0511584a0925f9c15}{class}} GRM$>$\newline
class dai\+::\+DAIAlg$<$ GRM $>$}
Combines the abstract base class \doxylink{classdai_1_1InfAlg}{Inf\+Alg} with a graphical model (e.\+g., a \doxylink{classdai_1_1FactorGraph}{Factor\+Graph} or \doxylink{classdai_1_1RegionGraph}{Region\+Graph}). 

Inference algorithms in lib\+DAI directly inherit from a \doxylink{classdai_1_1DAIAlg}{DAIAlg}, currently either from a DAIAlg$<$\+Factor\+Graph$>$ or from a DAIAlg$<$\+Region\+Graph$>$.


\begin{DoxyTemplParams}{Template Parameters}
{\em GRM} & Should be castable to \doxylink{classdai_1_1FactorGraph}{Factor\+Graph} \\
\hline
\end{DoxyTemplParams}
\begin{DoxyRefDesc}{Idea}
\item[\mbox{\hyperlink{ideas__ideas000005}{Idea}}]A \doxylink{classdai_1_1DAIAlg}{DAIAlg} should not inherit from a \doxylink{classdai_1_1FactorGraph}{Factor\+Graph} or \doxylink{classdai_1_1RegionGraph}{Region\+Graph}, but should store a reference to the graphical model object. This prevents needless copying of (possibly large) data structures. Disadvantage\+: the caller must not change the graphical model between calls to the inference algorithm (maybe a smart\+\_\+ptr or some locking mechanism would help here?). \end{DoxyRefDesc}
