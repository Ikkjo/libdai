\doxysubsection{Detailed Description}
Represents a set of properties, mapping keys (of type Property\+Key) to values (of type Property\+Value) 

Properties are used for specifying parameters of algorithms in a convenient way, where the values of the parameters can be of different types (e.\+g., strings, doubles, integers, enums). A \doxylink{classdai_1_1PropertySet}{Property\+Set} is an attempt to mimic the functionality of a Python dictionary object in C++, using the boost\+::any class.

A \doxylink{classdai_1_1PropertySet}{Property\+Set} can be converted to and from a string, using the following format\+:

{\ttfamily \mbox{[}key1=val1,key2=val2,...,keyn=valn\mbox{]}}

That is,
\begin{DoxyItemize}
\item the whole \doxylink{classdai_1_1PropertySet}{Property\+Set} is wrapped in square brackets ("{}\mbox{[}"{}, "{}\mbox{]}"{})
\item all properties in the \doxylink{classdai_1_1PropertySet}{Property\+Set} are seperated by a comma ("{},"{})
\item each Property consists of\+:
\begin{DoxyItemize}
\item the name of the key
\item an equality sign ("{}="{})
\item its value (represented as a string)
\end{DoxyItemize}
\end{DoxyItemize}

Also, a \doxylink{classdai_1_1PropertySet}{Property\+Set} provides functionality for converting the representation of individual values from some arbitrary type to and from std\+::string.

\begin{DoxyNote}{Note}
Not all types are automatically supported; if a type is unknown, an UNKNOWN\+\_\+\+PROPERTY\+\_\+\+TYPE exception is thrown. Adding support for a new type can be done in the body of the \doxylink{classdai_1_1PropertySet_ae7d3211d4902bb926110f9b2c8b1d99d}{operator$<$$<$(std\+::ostream \&, const Property \&)}. 
\end{DoxyNote}
